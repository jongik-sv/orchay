# 페이지 생성/삭제 기능 사용자 매뉴얼

**Task ID**: TSK-02-04
**버전**: 1.0
**작성일**: 2026-01-03

---

## 1. 개요

### 1.1 기능 소개
이 기능을 통해 사용자는 Notion 스타일의 페이지 구조를 자유롭게 관리할 수 있습니다:
- **새 페이지 생성**: 루트 레벨 또는 기존 페이지 하위에 새 페이지 추가
- **페이지 삭제**: 불필요한 페이지 제거 (하위 페이지 포함)

### 1.2 대상 사용자
- Orchay Notes 애플리케이션 사용자
- 페이지 계층 구조를 관리해야 하는 사용자

---

## 2. 시작하기

### 2.1 사전 요구사항
- Orchay Notes 애플리케이션이 실행 중이어야 합니다
- 브라우저에서 `http://localhost:3000` 접속

### 2.2 접근 방법
페이지 관리 기능은 사이드바를 통해 접근합니다:
1. 화면 왼쪽 사이드바 확인
2. 페이지 목록과 "New page" 버튼 확인

---

## 3. 사용 방법

### 3.1 새 페이지 생성

#### 3.1.1 루트 페이지 생성
1. 사이드바 하단의 **"New page"** 버튼 클릭
2. 새 페이지가 생성되고 자동으로 해당 페이지로 이동
3. 에디터에서 제목과 내용 작성 시작

#### 3.1.2 하위 페이지 생성
1. 사이드바에서 부모가 될 페이지 찾기
2. 페이지 항목을 **우클릭** 또는 **호버 후 더보기(⋯) 아이콘 클릭**
3. 컨텍스트 메뉴에서 **"Add subpage"** 선택
4. 하위 페이지가 생성되고 자동으로 해당 페이지로 이동
5. 부모 페이지 하위에 새 페이지가 트리 형태로 표시

### 3.2 페이지 삭제

#### 3.2.1 삭제 절차
1. 사이드바에서 삭제할 페이지 찾기
2. 페이지 항목을 **우클릭** 또는 **호버 후 더보기(⋯) 아이콘 클릭**
3. 컨텍스트 메뉴에서 **"Delete"** 선택
4. 확인 모달에서 경고 메시지 확인
5. **"Delete"** 버튼 클릭하여 삭제 확정

#### 3.2.2 주의사항
- **하위 페이지 포함 삭제**: 페이지 삭제 시 모든 하위 페이지도 함께 삭제됩니다
- **되돌리기 불가**: 삭제된 페이지는 복구할 수 없습니다
- **현재 페이지 삭제**: 보고 있던 페이지가 삭제되면 다른 페이지로 자동 이동합니다

### 3.3 상세 기능

#### 3.3.1 컨텍스트 메뉴
| 메뉴 항목 | 설명 |
|----------|------|
| Open | 해당 페이지 열기 |
| Add subpage | 하위에 새 페이지 생성 |
| Delete | 페이지 삭제 (확인 필요) |

#### 3.3.2 삭제 확인 모달
- **제목**: "Delete page?"
- **메시지**: 하위 페이지 유무에 따른 경고
  - 하위 페이지 있음: "이 페이지와 모든 하위 페이지가 삭제됩니다"
  - 하위 페이지 없음: "이 페이지가 삭제됩니다"
- **버튼**:
  - Cancel: 삭제 취소
  - Delete: 삭제 확정 (빨간색)

## 4. FAQ

### Q1: 삭제한 페이지를 복구할 수 있나요?
**A**: 현재 버전에서는 휴지통 기능이 없어 삭제된 페이지는 복구할 수 없습니다. 삭제 전 확인 모달에서 신중하게 결정해 주세요.

### Q2: 하위 페이지만 남기고 상위 페이지만 삭제할 수 있나요?
**A**: 아니요, 상위 페이지 삭제 시 모든 하위 페이지도 함께 삭제됩니다. 하위 페이지를 유지하려면 먼저 다른 위치로 이동(향후 지원 예정)하거나 내용을 복사해 두세요.

### Q3: 페이지를 드래그 앤 드롭으로 이동할 수 있나요?
**A**: 현재 MVP 버전에서는 드래그 앤 드롭 이동 기능이 지원되지 않습니다. 향후 업데이트에서 추가될 예정입니다.

### Q4: 생성할 수 있는 페이지 수에 제한이 있나요?
**A**: 별도의 제한은 없습니다. SQLite 데이터베이스의 용량 범위 내에서 자유롭게 페이지를 생성할 수 있습니다.

## 5. 문제 해결

### 5.1 새 페이지가 생성되지 않을 때
- **원인**: 서버 연결 문제 또는 데이터베이스 오류
- **해결**:
  1. 브라우저 개발자 도구 콘솔에서 에러 메시지 확인
  2. 개발 서버(`npm run dev`)가 실행 중인지 확인
  3. 브라우저 새로고침 후 재시도

### 5.2 삭제 버튼이 동작하지 않을 때
- **원인**: 네트워크 지연 또는 API 오류
- **해결**:
  1. 잠시 기다린 후 다시 시도
  2. 브라우저 새로고침 후 재시도
  3. 개발자 도구 Network 탭에서 요청 상태 확인

### 5.3 사이드바에 변경사항이 반영되지 않을 때
- **원인**: 상태 동기화 지연
- **해결**:
  1. 브라우저 새로고침
  2. 페이지 이동 후 다시 사이드바 확인
