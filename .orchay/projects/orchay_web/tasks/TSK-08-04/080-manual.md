## 1. 개요

### 1.1 작업 요약

AppHeader 컴포넌트를 PrimeVue Menubar로 마이그레이션하여 일관된 UI 컴포넌트 라이브러리를 사용하도록 개선했습니다.

**작업 범위**:
- 커스텀 네비게이션 → PrimeVue Menubar 교체
- CSS 클래스 중앙화 원칙 준수 (main.css 통합)
- 기존 기능 100% 유지 (회귀 없음)

### 1.2 변경 사항 요약

| 구분 | 변경 전 | 변경 후 |
|------|---------|---------|
| 메뉴 컴포넌트 | 커스텀 버튼 | PrimeVue Menubar |
| 스타일링 방식 | TailwindCSS 직접 사용 | Pass Through API + main.css |
| 메뉴 모델 | 하드코딩 배열 | MenuItem 모델 (computed) |
| 네비게이션 | 수동 라우팅 | PrimeVue to 속성 자동 라우팅 |

**결과**: 더 유지보수하기 쉽고, PrimeVue 생태계와 통합된 헤더 컴포넌트

## 2. 주요 기능

### 2.1 로고

**위치**: 헤더 왼쪽 끝

**기능**:
- "orchay" 텍스트 로고 표시
- 클릭 시 WBS 페이지로 이동

**시각적 특징**:
- Primary 색상 (#3b82f6)
- 마우스 오버 시 투명도 감소 효과

### 2.2 네비게이션 메뉴

**위치**: 로고 오른쪽, 4개 메뉴 항목

| 메뉴 | 경로 | 상태 | 클릭 동작 |
|------|------|------|----------|
| 대시보드 | /dashboard | 비활성 | "준비 중입니다" 토스트 표시 |
| 칸반 | /kanban | 비활성 | "준비 중입니다" 토스트 표시 |
| WBS | /wbs | **활성** | WBS 페이지로 이동 |
| Gantt | /gantt | 비활성 | "준비 중입니다" 토스트 표시 |

#### 2.2.1 활성 메뉴 (WBS)

**시각적 특징**:
- Primary 색상 (#3b82f6) 텍스트
- Primary 색상 배경 (20% 투명도)
- 현재 페이지임을 강조

**동작**:
- 클릭 시 /wbs 페이지로 이동
- 마우스 오버 시 호버 효과 표시

#### 2.2.2 비활성 메뉴 (대시보드, 칸반, Gantt)

**시각적 특징**:
- 회색 텍스트 (#666666)
- 50% 투명도 적용 (흐릿하게 표시)
- 마우스 커서가 "금지" 아이콘으로 변경

**동작**:
- 클릭 시 페이지 이동 없음
- "알림: 준비 중입니다" 토스트 메시지 표시 (3초 후 자동 닫힘)

### 2.3 프로젝트명 표시

**위치**: 헤더 오른쪽 끝

**표시 규칙**:
1. 프로젝트 선택됨: 프로젝트명 표시 (예: "orchay")
2. 프로젝트 미선택: "프로젝트를 선택하세요" (이탤릭체)

**시각적 특징**:
- 최대 200px 너비
- 텍스트가 길면 말줄임표(...) 표시
- 프로젝트명: 밝은 회색 (#888888)
- 안내 메시지: 어두운 회색 (#666666), 이탤릭체

## 3. 사용 방법

### 3.1 기본 네비게이션

**Step 1**: 원하는 메뉴 클릭
- 활성 메뉴 (WBS): 즉시 페이지 이동
- 비활성 메뉴: "준비 중입니다" 토스트 표시

**Step 2**: 로고 클릭으로 WBS 홈 복귀
- 언제든지 "orchay" 로고를 클릭하면 WBS 페이지로 돌아갑니다

### 3.2 현재 위치 확인

**시각적 표시**:
- 현재 페이지의 메뉴는 파란색(Primary) 배경으로 강조 표시됩니다
- 예: WBS 페이지에 있으면 "WBS" 메뉴가 파란색으로 표시

### 3.3 비활성 메뉴 사용

**준비 중인 메뉴**:
- 대시보드, 칸반, Gantt는 현재 개발 중입니다
- 클릭하면 "준비 중입니다" 알림이 표시됩니다
- 향후 업데이트에서 활성화될 예정입니다

## 4. 접근성 안내

### 4.1 키보드 네비게이션

orchay은 마우스 없이 키보드만으로도 네비게이션을 사용할 수 있습니다.

| 키 | 동작 |
|----|------|
| Tab | 다음 메뉴 항목으로 이동 |
| Shift + Tab | 이전 메뉴 항목으로 이동 |
| Enter | 현재 포커스된 메뉴 활성화 |
| Space | 현재 포커스된 메뉴 활성화 |

**사용 예시**:
1. Tab 키를 눌러 메뉴 항목으로 이동
2. Enter 또는 Space 키로 메뉴 선택
3. 비활성 메뉴를 선택하면 "준비 중입니다" 알림

### 4.2 스크린 리더 지원

**지원 기능**:
- ARIA landmark: 헤더 영역을 "banner" 역할로 식별
- ARIA 현재 페이지 표시: 활성 메뉴는 "현재 페이지"로 안내
- ARIA 비활성 상태 표시: 비활성 메뉴는 "비활성화된 링크"로 안내
- ARIA 라벨: 비활성 메뉴는 "(준비 중)" 안내 추가

**스크린 리더 예상 출력**:
```
"orchay, 홈으로 이동, 링크"
→ "대시보드 (준비 중), 비활성화된 링크"
→ "칸반 (준비 중), 비활성화된 링크"
→ "WBS, 현재 페이지"
→ "Gantt (준비 중), 비활성화된 링크"
→ "프로젝트를 선택하세요"
```

### 4.3 색상 대비

**WCAG 2.1 AA 수준 준수**:
- 활성 메뉴: Primary 색상 (#3b82f6)과 배경의 대비 4.5:1 이상
- 비활성 메뉴: 회색 텍스트 (#666666)와 배경의 대비 4.5:1 이상
- 프로젝트명: 밝은 회색 (#888888)과 배경의 대비 3:1 이상 (Large Text)

## 5. 기술적 개선 사항

### 5.1 사용자에게 보이는 변화

**향상된 점**:
- 더 일관된 UI (PrimeVue 컴포넌트 라이브러리 사용)
- 부드러운 애니메이션 (메뉴 호버 효과)
- 향상된 접근성 (ARIA 속성 강화)

**변화 없는 점**:
- 메뉴 개수 및 순서 동일 (4개 메뉴)
- 활성/비활성 상태 동일 (WBS만 활성)
- 클릭 동작 동일 (비활성 메뉴 토스트 표시)
- 프로젝트명 표시 규칙 동일

### 5.2 CSS 클래스 중앙화

**개선 전**:
- 인라인 스타일 및 HEX 코드 하드코딩
- 컴포넌트마다 스타일 분산

**개선 후**:
- 모든 스타일을 main.css에서 중앙 관리
- TailwindCSS 클래스 사용
- 유지보수성 향상

## 6. 문제 해결

### 6.1 비활성 메뉴가 클릭되지 않아요

**원인**: 비활성 메뉴는 의도적으로 페이지 이동을 차단합니다

**해결**:
- 정상 동작입니다
- "준비 중입니다" 토스트가 표시되면 해당 메뉴는 현재 개발 중입니다
- WBS 메뉴만 사용 가능합니다

### 6.2 프로젝트명이 "프로젝트를 선택하세요"로 표시돼요

**원인**: 프로젝트가 선택되지 않았습니다

**해결**:
1. 프로젝트 목록 페이지로 이동
2. 원하는 프로젝트 선택
3. 프로젝트명이 헤더에 표시됩니다

### 6.3 메뉴가 파란색으로 강조되지 않아요

**원인**: 현재 페이지와 메뉴가 일치하지 않습니다

**확인**:
- WBS 페이지에서는 "WBS" 메뉴가 파란색으로 표시됩니다
- 다른 페이지로 이동하면 해당 메뉴가 강조됩니다 (활성화된 경우)
