## 1. 개요

### 1.1 기능 소개
AppHeader는 orchay 애플리케이션의 상단 헤더 컴포넌트입니다. 로고, 네비게이션 메뉴, 현재 프로젝트명을 표시하며, 사용자가 애플리케이션 내에서 쉽게 탐색할 수 있도록 합니다.

### 1.2 대상 사용자
- orchay을 사용하는 모든 사용자
- 프로젝트 관리자
- 개발자

---

## 2. 시작하기

### 2.1 사전 요구사항
- orchay 애플리케이션 실행 (`npm run dev`)
- 웹 브라우저 (Chrome, Firefox, Edge 권장)
- 최소 화면 너비: 1200px

### 2.2 접근 방법
AppHeader는 모든 페이지 상단에 고정되어 있습니다. 별도의 접근 절차 없이 항상 표시됩니다.

---

## 3. 사용 방법

### 3.1 헤더 구조

```
┌─────────────────────────────────────────────────────────┐
│ [orchay]    [대시보드] [칸반] [WBS] [Gantt]    프로젝트명 │
│   로고           네비게이션 메뉴              프로젝트   │
└─────────────────────────────────────────────────────────┘
```

### 3.2 로고 영역

#### 기능
- **orchay** 텍스트 로고 표시
- 클릭 시 WBS 페이지(`/wbs`)로 이동

#### 사용법
1. 로고 "orchay" 클릭
2. WBS 페이지로 자동 이동

**팁**: 어느 페이지에서든 로고를 클릭하면 메인 화면(WBS)으로 돌아갈 수 있습니다.

---

### 3.3 네비게이션 메뉴

#### 메뉴 구성
| 메뉴 | 설명 | 활성화 상태 |
|------|------|-------------|
| 대시보드 | 프로젝트 전체 현황 | 비활성 (2차 개발) |
| 칸반 | 칸반 보드 | 비활성 (2차 개발) |
| WBS | WBS 트리 뷰 | **활성** |
| Gantt | Gantt 차트 | 비활성 (2차 개발) |

#### 활성 메뉴 사용
1. **WBS** 메뉴 클릭
2. `/wbs` 페이지로 이동
3. 현재 페이지에 해당하는 메뉴는 파란색으로 강조됨

#### 비활성 메뉴 클릭 시
1. 비활성 메뉴(대시보드, 칸반, Gantt) 클릭
2. "준비 중입니다" Toast 알림 표시
3. 페이지 이동 없음

**시각적 구분**:
- 활성 메뉴: 선명한 텍스트
- 비활성 메뉴: 흐린 텍스트 (opacity 50%)
- 현재 페이지: 파란색 배경 강조

---

### 3.4 프로젝트명 표시

#### 기능
- 현재 선택된 프로젝트명 표시
- 프로젝트 미선택 시 안내 텍스트 표시

#### 표시 상태
| 상태 | 표시 내용 |
|------|----------|
| 프로젝트 선택됨 | 프로젝트 이름 (예: "ORCHAY") |
| 프로젝트 미선택 | "프로젝트를 선택하세요" (이탤릭) |

## 4. 키보드 접근성

### 4.1 키보드 네비게이션
| 키 | 동작 |
|----|------|
| Tab | 다음 요소로 이동 (로고 → 메뉴들) |
| Shift + Tab | 이전 요소로 이동 |
| Enter | 현재 포커스된 요소 클릭 |

### 4.2 포커스 순서
1. 로고 (NuxtLink)
2. 대시보드 메뉴
3. 칸반 메뉴
4. WBS 메뉴
5. Gantt 메뉴

## 5. 자주 묻는 질문 (FAQ)

### Q1: 대시보드/칸반/Gantt 메뉴는 언제 사용할 수 있나요?
**A**: 2차 개발에서 활성화될 예정입니다. 현재는 WBS 메뉴만 사용 가능합니다.

### Q2: 프로젝트명이 표시되지 않아요
**A**: 프로젝트가 선택되지 않은 상태입니다. WBS 페이지에서 프로젝트를 선택하면 표시됩니다.

### Q3: 로고를 클릭해도 페이지가 이동하지 않아요
**A**: 이미 WBS 페이지에 있는 경우 동일한 페이지로 이동하므로 변화가 없을 수 있습니다.

### Q4: Toast 알림이 너무 빨리 사라져요
**A**: Toast 알림은 3초 후 자동으로 사라집니다. 알림 내용을 확인하셨다면 정상입니다.

## 6. 문제 해결

### 6.1 일반적인 문제

#### 문제: 메뉴가 보이지 않음
**원인**: 화면 너비가 1200px 미만
**해결 방법**: 브라우저 창을 넓히거나 전체 화면 모드 사용

#### 문제: 프로젝트명이 잘림
**원인**: 프로젝트명이 200px를 초과
**해결 방법**: 정상 동작입니다. 긴 이름은 말줄임표(...)로 표시됩니다.

#### 문제: Toast 알림이 표시되지 않음
**원인**: PrimeVue ToastService 초기화 문제
**해결 방법**: 페이지 새로고침 후 다시 시도
