## 1. 개요

### 1.1 목적

orchay 애플리케이션의 상단 헤더 영역을 담당하는 AppHeader 컴포넌트를 구현합니다. 로고, 네비게이션 메뉴, 현재 프로젝트명을 표시하여 사용자가 앱 내에서 쉽게 탐색할 수 있도록 합니다.

### 1.2 구현 범위

> WBS Task 설명에서 추출

- orchay 로고
- 네비게이션 메뉴 (대시보드, 칸반, WBS, Gantt) - WBS만 활성
- 현재 프로젝트명 표시

### 1.3 제외 범위

> 동일 PRD 섹션이지만 다른 Task에서 구현

- 대시보드 페이지 → 2차 개발
- 칸반 보드 페이지 → 2차 개발
- Gantt 차트 페이지 → 2차 개발
- 프로젝트 선택 드롭다운 → 2차 개발

---

## 2. 사용자 시나리오

### 2.1 주요 사용자

- **프로젝트 매니저**: 다양한 뷰 간 전환
- **개발자**: WBS 페이지에서 작업 확인
- **모든 사용자**: 현재 프로젝트 확인

### 2.2 사용 시나리오

**시나리오 1: 앱 접속**
1. 사용자가 orchay 앱에 접속
2. 헤더에 orchay 로고와 네비게이션 메뉴 표시
3. 현재 활성화된 메뉴(WBS)가 강조 표시됨
4. 우측에 현재 프로젝트명 표시

**시나리오 2: 메뉴 클릭 (1차에서는 WBS만)**
1. 사용자가 WBS 메뉴 클릭
2. WBS 페이지로 이동
3. WBS 메뉴가 활성 상태로 표시

**시나리오 3: 비활성 메뉴 클릭 (1차)**
1. 사용자가 대시보드/칸반/Gantt 메뉴 클릭
2. "준비 중" 토스트 메시지 표시
3. 페이지 이동 없음

## 3. 기능 요구사항

> PRD 6.1에서 범위 내 항목만 추출

### 3.1 로고 표시

**설명**: 앱 브랜드 아이덴티티 제공
**입력**: -
**출력**: orchay 로고 (텍스트 또는 이미지)
**제약조건**:
- 클릭 시 홈(WBS) 페이지로 이동
- Primary 색상 강조

### 3.2 네비게이션 메뉴

**설명**: 주요 기능 페이지 간 이동
**입력**: 사용자 클릭
**출력**: 해당 페이지로 라우팅
**제약조건**:
- 1차 범위: WBS만 활성화
- 비활성 메뉴: disabled 상태로 표시
- 현재 페이지 메뉴 강조 표시

### 3.3 프로젝트명 표시

**설명**: 현재 작업 중인 프로젝트 식별
**입력**: 프로젝트 정보 (Pinia store)
**출력**: 프로젝트명 텍스트
**제약조건**:
- 프로젝트 미선택 시 "프로젝트를 선택하세요" 표시
- 텍스트 오버플로우 시 말줄임 처리
