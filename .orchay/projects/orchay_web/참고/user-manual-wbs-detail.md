# 사용자 매뉴얼: WBS 트리 & Task 상세 패널

**버전:** 1.0.0 — **최종 수정일:** 2025-12-16

## 1. 개요

orchay의 프로젝트 관리 화면은 **좌측 WBS 트리 패널**과 **우측 Task 상세 패널**로 구성됩니다. 이 매뉴얼은 두 패널을 활용한 주요 사용자 시나리오를 안내합니다.

### 1.1 화면 구성

```
┌─────────────────────────────────────────────────────────────┐
│  Header                                                      │
├──────────────────────┬──────────────────────────────────────┤
│  WBS 트리 패널 (40%)  │  Task 상세 패널 (60%)                 │
│  ┌────────────────┐  │  ┌────────────────────────────────┐  │
│  │ 검색           │  │  │ 기본 정보                       │  │
│  │ [통계 카드]    │  │  │  - ID, 제목, 카테고리...        │  │
│  │ [전체 펼치기]  │  │  ├────────────────────────────────┤  │
│  ├────────────────┤  │  │ 진행 상태                       │  │
│  │ Project        │  │  │  - 워크플로우 시각화            │  │
│  │  └ WP-01       │  │  └────────────────────────────────┘  │
│  │    └ TSK-01-01 │  │                                      │
│  └────────────────┘  │                                      │
└──────────────────────┴──────────────────────────────────────┘
```

### 1.2 주요 기능 요약

| 패널 | 기능 |
|------|------|
| WBS 트리 | 프로젝트 구조 탐색, 검색, 통계 확인 |
| Task 상세 | 정보 조회, 인라인 편집, 워크플로우 확인 |

## 2. 시나리오별 사용 가이드

### 시나리오 1: 프로젝트 현황 파악하기

**목표**: 프로젝트의 전체 WBS 구조와 진행 현황을 한눈에 파악합니다.

#### 단계별 안내

**Step 1.** 프로젝트 페이지에 진입하면 좌측에 WBS 트리가 자동 로드됩니다.

**Step 2.** 상단 **통계 카드**에서 현황을 확인합니다.

| 카드 | 의미 | 색상 |
|------|------|------|
| WP | Work Package 수 | 파란색 |
| ACT | Activity 수 | 녹색 |
| TSK | Task 수 | 주황색 |
| % | 전체 진행률 | 보라색 |

**Step 3.** 트리 노드를 클릭하여 계층 구조를 탐색합니다.
- `>` 클릭: 하위 노드 펼치기
- `v` 클릭: 하위 노드 접기

**Step 4.** 전체 구조를 보려면 **[전체 펼치기]** 버튼을 클릭합니다.

> **Tip**: 많은 Task가 있을 경우 **[전체 접기]**로 최상위만 표시하여 정리할 수 있습니다.

---

### 시나리오 2: 특정 Task 빠르게 찾기

**목표**: 검색 기능으로 원하는 Task를 빠르게 찾습니다.

#### 단계별 안내

**Step 1.** 검색 박스를 클릭합니다.

**Step 2.** Task ID 또는 제목의 일부를 입력합니다.
- 예: `TSK-04` 또는 `Tree Panel`

**Step 3.** 입력 후 약 0.3초 대기하면 자동 필터링됩니다.
- 매칭된 노드가 하이라이트 표시됩니다.

**Step 4.** 원하는 Task를 클릭하여 상세 정보를 확인합니다.

#### 검색 조작법

| 조작 | 동작 |
|------|------|
| 입력 | 300ms 후 자동 검색 |
| **X** 버튼 | 검색어 초기화 |
| **ESC** 키 | 검색어 초기화 |

> **주의**: 검색은 Task ID와 제목만 대상으로 합니다. 내용 검색은 지원하지 않습니다.

---

### 시나리오 3: Task 상세 정보 확인하기

**목표**: 선택한 Task의 기본 정보와 진행 상태를 확인합니다.

#### 단계별 안내

**Step 1.** WBS 트리에서 Task 노드를 클릭합니다.
- WP나 ACT 클릭 시 안내 메시지가 표시됩니다.

**Step 2.** 우측 패널에 로딩 후 상세 정보가 표시됩니다.

**Step 3.** **기본 정보** 섹션에서 확인합니다:

| 필드 | 설명 | 편집 가능 |
|------|------|----------|
| ID | Task 고유 식별자 | X |
| 제목 | Task 이름 | O |
| 카테고리 | development/defect/infrastructure | O |
| 우선순위 | critical/high/medium/low | O |
| 담당자 | 배정된 팀원 | O |

**Step 4.** **진행 상태** 섹션에서 워크플로우 위치를 확인합니다.

```
워크플로우 시각화 예시 (development 카테고리):

  [v]━━━━[*]━━━━[ ]━━━━[ ]━━━━[ ]━━━━[ ]
 Todo  Design Detail Impl Verify Done
         ^ 현재 단계
```

| 표시 | 의미 |
|------|------|
| [v] (녹색) | 완료된 단계 |
| [*] (파란색) | 현재 단계 |
| [ ] (회색) | 대기 중 단계 |

---

### 시나리오 4: Task 정보 수정하기

**목표**: 인라인 편집으로 Task 정보를 즉시 수정합니다.

#### 4-A. 제목 편집

**Step 1.** 제목 필드에 마우스를 올리면 편집 아이콘이 나타납니다.

**Step 2.** 클릭하면 입력 필드로 전환됩니다.

**Step 3.** 새 제목을 입력합니다 (1~200자).

**Step 4.** 저장 또는 취소:

| 조작 | 결과 |
|------|------|
| **Enter** | 저장 |
| **ESC** | 취소 (원래 값 복원) |
| 외부 클릭 | 취소 |

> 저장 시 UI가 즉시 반영됩니다 (낙관적 업데이트).

#### 4-B. 카테고리/우선순위/담당자 변경

**Step 1.** 해당 필드를 클릭합니다.

**Step 2.** Dropdown 목록에서 선택합니다.

**Step 3.** 선택 즉시 자동 저장됩니다.

##### 카테고리 옵션

| 값 | 표시 | 워크플로우 단계 |
|----|------|----------------|
| development | 파란색 배지 | 6단계 |
| defect | 빨간색 배지 | 5단계 |
| infrastructure | 보라색 배지 | 4단계 |

##### 우선순위 옵션

| 값 | 표시 |
|----|------|
| critical | 긴급 (빨간색) |
| high | 높음 (주황색) |
| medium | 보통 (파란색) |
| low | 낮음 (회색) |

> **주의**: 카테고리 변경 시 워크플로우가 재계산됩니다.

---

### 시나리오 5: 카테고리별 워크플로우 이해하기

**목표**: 각 카테고리의 워크플로우 단계를 이해합니다.

#### Development (개발) - 6단계

```
Todo [ ] -> Design [bd] -> Detail [dd] -> Implement [im] -> Verify [vf] -> Done [xx]
```

| 단계 | 코드 | 설명 |
|------|------|------|
| Todo | `[ ]` | 작업 대기 |
| Design | `[bd]` | 기본설계 |
| Detail | `[dd]` | 상세설계 |
| Implement | `[im]` | 구현 |
| Verify | `[vf]` | 통합테스트 |
| Done | `[xx]` | 완료 |

#### Defect (결함) - 5단계

```
Todo [ ] -> Analyze [an] -> Fix [fx] -> Verify [vf] -> Done [xx]
```

| 단계 | 코드 | 설명 |
|------|------|------|
| Todo | `[ ]` | 결함 접수 |
| Analyze | `[an]` | 원인 분석 |
| Fix | `[fx]` | 수정 |
| Verify | `[vf]` | 검증 |
| Done | `[xx]` | 완료 |

#### Infrastructure (인프라) - 4단계

```
Todo [ ] -> Design [ds] -> Implement [im] -> Done [xx]
```

| 단계 | 코드 | 설명 |
|------|------|------|
| Todo | `[ ]` | 작업 대기 |
| Design | `[ds]` | 설계 (선택) |
| Implement | `[im]` | 구현 |
| Done | `[xx]` | 완료 |

## 5. FAQ / 트러블슈팅

### Q1. 검색해도 결과가 안 나와요.
- Task ID와 제목만 검색 대상입니다.
- 대소문자 구분 없이 검색됩니다.
- 검색어를 초기화하고 다시 시도해 보세요.

### Q2. 편집 내용이 저장되지 않아요.
- 네트워크 연결을 확인하세요.
- 제목은 1~200자 제한이 있습니다.
- 에러 메시지를 확인하고 재시도하세요.

### Q3. 워크플로우가 안 보여요.
- Task 타입 노드만 워크플로우가 표시됩니다.
- WP나 ACT 선택 시에는 표시되지 않습니다.

### Q4. 담당자 목록이 비어있어요.
- 프로젝트 설정에서 팀원을 먼저 등록해야 합니다.
- `team.json` 파일을 확인하세요.

## 6. 접근성

### 6.1 지원 기능

- 모든 버튼과 입력 필드에 `aria-label` 제공
- 검색 박스: `role="searchbox"`
- 로딩 상태: `aria-busy="true"`
- 상태 변경 시 스크린 리더 음성 안내 (`aria-live`)
- 키보드만으로 모든 기능 조작 가능

### 6.2 색상 대비

모든 텍스트는 WCAG AA 기준 (4.5:1) 이상의 명암비를 충족합니다.
